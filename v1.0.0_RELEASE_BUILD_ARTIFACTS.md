## Known Issues & Resolutions

### ⚠️ Issue 1: AAB Build Symbol Stripping
**Error Message:**
```
Release app bundle failed to strip debug symbols from native libraries
```

**Root Cause:** Missing or misconfigured NDK in Android SDK

**Resolution Options:**
1. Install/verify NDK via Android Studio SDK Manager
2. Set NDK path in `android/local.properties`:
   ```properties
   ndk.dir=/Users/vinayakballary/Library/Android/sdk/ndk/27.0.12077973
   ```
3. Run: `flutter doctor -v` and address any issues
4. Retry: `flutter build appbundle --release`

**Workaround:** Use APK for testing; AAB required for production Play Store upload

### ⚠️ Issue 2: Code Minification Disabled
**Reason:** R8 missing Play Store Core dependencies (com.google.android.play.core.*)

**Impact:**
- APK size: 52 MB (vs ~35 MB if minified)
- No code obfuscation (but app logic is still secure)

**Solutions:**
1. Add missing dependencies to build.gradle:
   ```gradle
   implementation 'com.google.android.play:core:1.10.3'
   ```
2. Update ProGuard rules for Play Core classes
3. Re-enable minification

### ⚠️ Issue 3: iOS IPA Not Built
**Reason:** iOS app configuration not verified in this build cycle

**Status:** Requires separate iOS provisioning setup
- Apple Developer account
- Provisioning profiles
- Signing certificates
- App Store Connect setup

**Resolution:** Run separate iOS build after provisioning: `flutter build ios --release`
