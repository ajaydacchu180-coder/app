## Rollback Plan

### If Critical Issues Found

**Immediate Actions (< 1 hour):**
1. [ ] Pause production rollout at current percentage
2. [ ] Notify support team
3. [ ] Collect crash details
4. [ ] Create incident ticket

**Short-term (1-4 hours):**
1. [ ] Identify root cause
2. [ ] Build v1.0.1 hotfix
3. [ ] Test fix thoroughly
4. [ ] Create git tag v1.0.1

**Medium-term (4-24 hours):**
1. [ ] Upload v1.0.1 to Play Console
2. [ ] Resume rollout with fixed version
3. [ ] Retire v1.0.0 from production

**Example Hotfix Scenarios:**
- **API authentication broken** → Fix API endpoint/token handling
- **Crash on startup** → Fix app initialization
- **Data loss** → Revert database changes
- **Performance degradation** → Optimize heavy operations
